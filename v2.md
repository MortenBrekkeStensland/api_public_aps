## APS API v2

API v2 is supported by APS 4.0 and later. Fields and commands added after APS 4.0 are marked with their minimum version where relevant. Clients should tolerate unknown fields and missing optional fields so they remain compatible across APS releases and platforms.


### Protocol:
APS has a TCP server that waits for a client connection on a default port (31600) and can be configured from the app settings. Once a connection is instantiated between the client and APS it will continuously listen to the incoming messages from that client.
APS uses length-prefixed JSON messages for all sent/received API traffic. API v1 is no longer supported in current APS versions.

For sending commands
1. Prepare the command as JSON string.
2. Calculate the length of the entire message.
3. Send the length of the entire message in 4 bytes big-endian.
4. Send the JSON string message.

Clients can optionally send an API version message to indicate their supported version:
```JSON
{
   "command": "api_version", 
   "api_version": 2
}
```
APS also will send the API version once the connection instantiated
```JSON
{
   "action": "api_version", 
   "api_version": 2
}
```
APS also sends app metadata once the connection is established:
```JSON
{
   "action": "aps_info",
   "data": {
      "app_version": "4.2.1.5",
      "computer_tag": "FOH-PC"
   }
}
```
APS 4.2 and later on macOS also includes the current app settings in the full state snapshot sent after connection. See [Settings](#settings) for the payload and the writable subset.

**Notes**

1. All the above messages are length-prefixed.
2. APS expects the API v2 protocol for client communication. If the `api_version` command is omitted, APS still uses the v2 protocol.
3. The `api_version` message is used to report version compatibility between APS and the connected client.



### Commands:

#### Commands structure
```JSON
{
   "command": "...", 
   "parameters": {
      "param1": "...",
      "param2": "...",
   }
}
```
Example:
```JSON
{
   "command": "OpenStart_Presentation", 
   "parameters": {
      "file_path": "C:\\1.pptx",
      "slideNr": 1,
      "isFullscreen": true
   }
}
```

#### Presentations file commands
1. `Navigation_NextFS`: Open the next presentation using the focused app in fullscreen
   - Parameters:
      - slideNr (int)
2. `Navigation_PrevFS`: Open prev presentation using the focused app in fullscreen
   - Parameters:
      - slideNr (int)
3. `Navigation_NextNoFS`: Open the next presentation using the focused app
4. `Navigation_CurrentFS`: Set current presentation in fullscreen
5. `OpenStart_Presentation`: open presentation by path
   - Parameters:
      - file_path (string)
      - slideNr (int)
      - isFullscreen (bool)
6. `OpenStart_Presentation_Slot`: open presentation by slot number
   - Parameters:
      - slot (int)
      - slideNr (int)
      - isFullscreen (bool)
7. `Navigation_CloseOthers`
8. `CapturePresentationSlot`: Capture current focused presentation and save its path to a slot
   - Parameters:
      - bank_number (int)
9. `SetPresentationSlotPath`: Update slot path by a given full path
   - Parameters:
      - slot (int)
      - file_path (string)

#### Presentations slide commands
1. `Key_Left`: Go to the previous slide in the active presentation application.
2. `Key_Right`: Go to the next slide in the active presentation application.
3. `Key_Esc`: Exit presentation/fullscreen mode in the active presentation application.
4. `Key_B`: Toggle the presentation black screen where supported.
5. `Generic`: Go to a specified slide in the active presentation application.
   - Parameters:
      - slideNr (int)
6. `Powerpoint_Go`: Go to a specified slide (PowerPoint)
   - Parameters:
      - slideNr (int)
7. `Powerpoint_Section_Go`: Go to the first slide of a specified 1-based PowerPoint section. Available in APS 4.2 and later on macOS.
   - Parameters:
      - section (int): The 1-based section index. It must be greater than zero.
8. `Powerpoint_Section_Name_Go`: Go to the first slide of a PowerPoint section with a specified name. Available in APS 4.2 and later on macOS.
   - Parameters:
      - section_name (string): The section name. APS trims outer whitespace and matches names case-insensitively. If multiple sections have the same name, the first section in presentation order is used.
9. `Powerpoint_Section_Previous`: Go to the first slide of the section before the current PowerPoint section. Available in APS 4.2 and later on macOS. If the current slide is in the first section, no navigation occurs.
10. `Powerpoint_Section_Next`: Go to the first slide of the section after the current PowerPoint section. Available in APS 4.2 and later on macOS. If the current slide is in the last section, no navigation occurs.
11. `Powerpoint_Previous`
12. `Powerpoint_Next`
13. `Acrobat_Go`: Go to a specified slide (Adobe Acrobat DC Pro)
   - Parameters:
      - slideNr (int)
14. `Acrobat_Previous`
15. `Acrobat_Next`
16. `Keynote_GO` Go to a specified slide (Keynote)
   - Parameters:
      - slideNr (int)
17. `Keynote_Previous`
18. `Keynote_Next`

PowerPoint section commands do nothing when PowerPoint is not running, no presentation is open, section information is unavailable, or the requested target section does not exist or is empty. Previous/next section navigation never wraps beyond the first or last section.

The generic slide commands work with the presentation application APS is currently controlling, including PowerPoint, Keynote, supported PDF applications, and Google Slides. Google Slides uses the existing generic commands; it does not require separate Google Slides command names. Google Slides control and slide feedback require APS 4.2 or later.

#### Web Browser Commands (Chrome)

1. `OpenWebPage`: Launches the APS-controlled Google Chrome instance with remote debugging enabled and an isolated APS browser profile.
   - Parameters:
      - url (string): The URL to open.
      - fullscreen (bool): If `true`, the browser opens in fullscreen mode.
      - new_tab (bool): If `true`, opens in a new tab; otherwise, reuses an existing tab.
2. `CloseWebBrowser`: Closes the Chrome instance started by `OpenWebPage`.
3. `SwitchTab`: Activates a specific browser tab using its remote debugging ID.
   - Parameters:
     - tabId (string): The debugging ID of the tab (e.g., `6689AD7376BBF72D319C4F5D4E16F10E`).
       You can obtain tab IDs from the [Webpage tabs feedback](#webpage-tabs).
4. `DeleteTab`: Closes a specific browser tab using its debugging ID.
   - Parameters:
     - tabId (string): The debugging ID of the tab (e.g., `6689AD7376BBF72D319C4F5D4E16F10E`).
       You can obtain tab IDs from the [Webpage tabs feedback](#webpage-tabs).
5. `RefreshTab`: Reloads a specific browser tab using its debugging ID.
   - Parameters:
     - tabId (string): The debugging ID of the tab (e.g., `6689AD7376BBF72D319C4F5D4E16F10E`).
       You can obtain tab IDs from the [Webpage tabs feedback](#webpage-tabs).
       Use `all` to reload all eligible browser tabs, equivalent to the app's `Ctrl+Shift+Alt+6` refresh-all shortcut.
6. `RefreshCurrentTab`: Reloads the currently active browser tab.
7. `CloseBrowserAndClearAllData`: Closes the browser and clears **all** browsing data, including sessions, cookies, history, and local storage.

#### General Commands
1. `Clear`: Clear slot or bank content
   - Parameters:
      - bank_number (string | int): can be "All" or integer
      - clear_type_key (string): can be one of these values [StillImages, Media, SlotPresentations, PresentationFolders, MediaFolders]
2. `ActivateApplication`: Brings the selected application to the foreground and activates its window.
   - Parameters:
      - application (string): it can be PowerPoint, PDF, or Webpage
3. `GetSettings`: Requests the current APS settings and installed presentation-app status. Available in APS 4.2 and later on macOS.
   - Parameters: none
   - Response: one [`settings`](#settings) feedback message
4. `SetSetting`: Sets one writable APS setting to an explicit value. Available in APS 4.2 and later on macOS.
   - Parameters:
      - `setting` (string): A writable setting name from the [Settings](#settings) table.
      - `value` (bool | string | object): The exact JSON type and allowed value documented for that setting.
   - Response: one [`settings_update_result`](#settings-update-result) message followed by one authoritative [`settings`](#settings) message.
5. `ToggleSetting`: Toggles one Boolean APS setting. Available in APS 4.2 and later on macOS.
   - Parameters:
      - `setting` (string): One of `seamless_switching`, `toggle_images_on_off_with_one_button`, `powerpoint_hide_presenter`, `google_slides_use_presenter_view`, or `automatically_check_for_updates`.
   - Do not include `value`. Enum and display settings must use `SetSetting`.
   - Response: one [`settings_update_result`](#settings-update-result) message followed by one authoritative [`settings`](#settings) message.

Example explicit write:

```json
{
  "command": "SetSetting",
  "parameters": {
    "setting": "seamless_switching",
    "value": true
  }
}
```

Example one-button toggle:

```json
{
  "command": "ToggleSetting",
  "parameters": {
    "setting": "seamless_switching"
  }
}
```

### Feedback:
Once the client gets connected, APS will continuously send info in length-prefixed JSON string.
> Some feedbacks will be sent only after getting triggered by commands like `capture`, `display`

#### Feedback structure
```JSON
{
   "action": "...",
   "data":{
      "data_key1": "...",
      "data_key2": [],
   }
}
```
- action (string): for specifying the type of this feedback
- data (object): it can containes any key-value pairs as required by the feedback, values can be any data type.

---

#### Presentations Slots Feedback
```JSON
{
   "action": "slots",
   "data":{
      "filenames": [ "01.pptx", "02.pptx", "03.pptx", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-", "-"],
      "exists": [true, true, true, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false],
      "opened": [true, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false, false]
   }
}
```

- `filenames`: An array of configured files in the `Presentations` tab
- `exists`:  An array to indicate if the file exists on the system or not
- `opened`:  An array to indicate if the file is currently opened or not

#### Files Feedback
```JSON
{
   "action": "files",
   "data":{
      "curr": "02.pptx",
      "prev": "01.pptx",
      "next":"-",
      "slide_number": "1",
      "slides_count": "10",
      "presenter_notes": "Welcome the audience and introduce the first topic.",
      "current_build": "0",
      "builds_count": "2",
      "powerpoint_slide_number": "1",
      "powerpoint_slides_count": "10",
      "powerpoint_current_build": "0",
      "powerpoint_builds_count": "2",
      "PowerPoint_media_duration": "00:19:50",
      "PowerPoint_media_current_position": "00:00:05",
      "PowerPoint_media_time_left": "00:19:45",
      "PowerPoint_media_state": "playing"
   }
}
```

- `curr`, `prev`, and `next`: Current, previous, and next presentation filenames. A value of `-` means no file is available.
- `slide_number` and `slides_count`: Current slide and total slide count for the presentation application APS is currently controlling.
- `presenter_notes`: Presenter notes for the current slide. Available in APS 4.2 and later on macOS when APS can read notes from PowerPoint, Keynote, or Google Slides. An empty string or `-` means no notes are available.
- `current_build` and `builds_count`: Current and total PowerPoint animation/build positions where available. A value of `-` means the information is unavailable.
- `powerpoint_slide_number`, `powerpoint_slides_count`, `powerpoint_current_build`, and `powerpoint_builds_count`: PowerPoint-specific state. These allow clients to retain PowerPoint state separately when another supported presentation application is active. These fields may be absent on older APS versions or unsupported platforms.

The `files` feedback is sent when presentation state changes. In APS 4.2 and later, Google Slides uses the same `slide_number`, `slides_count`, and `presenter_notes` fields as the other presentation applications.

Fields specific to PowerPoint media:

> **Note:** These fields are only available when controlling PowerPoint on Windows.

- `PowerPoint_media_duration`: Total media duration (hh:mm:ss).
- `PowerPoint_media_current_position`: Current playback position (hh:mm:ss).
- `PowerPoint_media_time_left`: Remaining time (hh:mm:ss).
- `PowerPoint_media_state`: One of `playing`, `paused`, `stopped`, or `not_ready`.

#### PowerPoint Sections Feedback

Available in APS 4.2 and later on macOS. This feedback describes the sections in the PowerPoint presentation currently running as a slideshow.

```JSON
{
   "action": "powerpoint_sections",
   "data": {
      "available": true,
      "current_section_index": 2,
      "current_section_id": "id-2",
      "current_section_name": "Main section",
      "current_slide_in_section": 1,
      "sections_count": 2,
      "sections": [
         {
            "id": "id-1",
            "name": "Default Section",
            "first_slide_index": 1,
            "slides_count": 2
         },
         {
            "id": "id-2",
            "name": "Main section",
            "first_slide_index": 3,
            "slides_count": 2
         }
      ]
   }
}
```

- `available`: `true` when APS has section information for the active PowerPoint slideshow. An available presentation may legitimately contain zero sections.
- `current_section_index`: The current 1-based section index, or `-1` when the displayed slide cannot be mapped to a section.
- `current_section_id`: PowerPoint's opaque identifier for the current section, or `-` when unavailable.
- `current_section_name`: The current section name, or `-` when unavailable.
- `current_slide_in_section`: The displayed slide's 1-based position within its section, or `-1` when unavailable.
- `sections_count`: The number of section records in `sections`.
- `sections`: Presentation-ordered PowerPoint section records. A record's 1-based position in the array is its section index. Each record contains its opaque `id`, `name`, absolute 1-based `first_slide_index`, and `slides_count`.

When section information is unavailable, APS sends `available: false`, an empty `sections` array, `sections_count: 0`, `-1` for unavailable numeric current fields, and `-` for the current section ID and name. This clears stale client state when PowerPoint closes or leaves slideshow mode.

If APS cannot read any section record completely, it reports section information as unavailable instead of sending a partial section list.

The feedback is sent when section metadata, the current section position, or PowerPoint slideshow availability changes. It is also included in the full state snapshot sent after a client connects.

#### Settings

Available in APS 4.2 and later on macOS. This feedback exposes the current Settings-tab choices, selectable displays, and detected presentation applications. A safe subset can be changed with `SetSetting` and `ToggleSetting`.

APS sends this feedback:

- as part of the full state snapshot when a client connects;
- in response to the parameterless `GetSettings` command;
- after a setting changes locally in APS;
- after every recognized remote settings write attempt, including failures and no-op writes; and
- when effective display or startup state changes, or the installed-app list is refreshed.

```json
{
  "action": "settings",
  "data": {
    "main_presenter_screen": {
      "selection": "automatic",
      "configured_display_id": null,
      "effective_display_id": 1,
      "effective_display_name": "Built-in Retina Display",
      "available_displays": [
        {
          "display_id": 1,
          "display_name": "Built-in Retina Display"
        },
        {
          "display_id": 8,
          "display_name": "Stage Display"
        }
      ]
    },
    "presentation_file_handling": "automatic",
    "seamless_switching": false,
    "run_at_system_startup": {
      "enabled": false,
      "status": "disabled"
    },
    "toggle_images_on_off_with_one_button": true,
    "powerpoint_hide_presenter": false,
    "google_slides_use_presenter_view": true,
    "pdf_controlled_program": "skim",
    "network_port": 31600,
    "automatically_check_for_updates": true,
    "installed_presentation_apps": [
      "powerpoint",
      "keynote",
      "adobe_acrobat",
      "skim"
    ]
  }
}
```

- `main_presenter_screen.selection`: `automatic` or `specific`.
- `configured_display_id`: The configured macOS display ID for a specific selection, or `null` for automatic selection.
- `effective_display_id` and `effective_display_name`: The display APS currently resolves for presentation output. Both can be `null` if no display is available. Display IDs describe the current Mac and should not be treated as portable identifiers.
- `available_displays`: The currently selectable displays in APS order. Each entry contains `display_id` and `display_name`. Use one of these IDs when selecting a specific presenter screen.
- `presentation_file_handling`: `automatic` or `controlled`.
- `seamless_switching`: Whether Seamless Switching is enabled.
- `run_at_system_startup.enabled`: `true` only when the macOS login item is enabled.
- `run_at_system_startup.status`: `enabled`, `disabled`, `requires_approval`, or `unavailable`.
- `toggle_images_on_off_with_one_button`: Whether the same still-image action toggles an image on and off.
- `powerpoint_hide_presenter`: Whether APS suppresses PowerPoint Presenter View.
- `google_slides_use_presenter_view`: Whether APS uses Google Slides Presenter View.
- `pdf_controlled_program`: `skim`, `adobe_acrobat`, or `none`.
- `network_port`: The configured APS TCP port. Changing it locally restarts the server and disconnects clients; reconnect on the new port to receive the new snapshot.
- `automatically_check_for_updates`: Whether APS automatically checks for updates.
- `installed_presentation_apps`: A fixed-order list filtered to detected apps. Possible values are `powerpoint`, `keynote`, `adobe_acrobat`, and `skim`.

The following settings are writable. Setting names and enum values are case-sensitive, and APS does not coerce strings or numbers into Booleans.

| Setting | `SetSetting` value | `ToggleSetting` | Validation |
| --- | --- | --- | --- |
| `main_presenter_screen` | Object; see below | No | A specific display ID must appear in `available_displays`. |
| `presentation_file_handling` | `automatic` or `controlled` | No | — |
| `seamless_switching` | Boolean | Yes | Enabling without Screen Recording permission succeeds with a warning and does not open local permission UI. |
| `toggle_images_on_off_with_one_button` | Boolean | Yes | — |
| `powerpoint_hide_presenter` | Boolean | Yes | PowerPoint must appear in `installed_presentation_apps`. |
| `google_slides_use_presenter_view` | Boolean | Yes | — |
| `pdf_controlled_program` | `skim` or `adobe_acrobat` | No | The selected app must be installed. `none` is feedback-only. |
| `automatically_check_for_updates` | Boolean | Yes | — |

Select the presenter screen automatically:

```json
{
  "command": "SetSetting",
  "parameters": {
    "setting": "main_presenter_screen",
    "value": {
      "selection": "automatic"
    }
  }
}
```

Select a specific currently available display:

```json
{
  "command": "SetSetting",
  "parameters": {
    "setting": "main_presenter_screen",
    "value": {
      "selection": "specific",
      "configured_display_id": 8
    }
  }
}
```

For convenience, clients may copy the complete `main_presenter_screen` feedback object into `value`; APS ignores its read-only effective-display and available-display fields.

The following settings and fields remain read-only over LAN:

- `network_port`: Change it locally in APS. A local port change restarts the server and disconnects clients.
- `run_at_system_startup`: Change it locally because macOS may require interactive approval.
- `installed_presentation_apps`: Detected by APS.
- `effective_display_id` and `effective_display_name`: Resolved by APS.

Attempts to write these fields return `read_only_setting`. Settings feedback may be absent on older APS versions or unsupported platforms.

#### Settings update result

APS sends this message first after every recognized `SetSetting` or `ToggleSetting` request. It then sends a complete [`settings`](#settings) message containing the authoritative state, regardless of whether the write succeeded, failed, or made no change.

Successful change:

```json
{
  "action": "settings_update_result",
  "data": {
    "success": true,
    "operation": "set",
    "setting": "seamless_switching",
    "changed": true,
    "warnings": [],
    "error": null
  }
}
```

An explicit value that already matches the current state is successful with `changed: false`. `ToggleSetting` uses `operation: "toggle"`.

Failure:

```json
{
  "action": "settings_update_result",
  "data": {
    "success": false,
    "operation": "set",
    "setting": "network_port",
    "changed": false,
    "warnings": [],
    "error": {
      "code": "read_only_setting",
      "message": "network_port can only be changed locally in APS."
    }
  }
}
```

- `success`: Whether APS accepted and applied the request.
- `operation`: `set` or `toggle`.
- `setting`: The requested setting name, or `null` when no valid string was supplied.
- `changed`: `true` only when the stored setting changed.
- `warnings`: Stable warning codes. Currently `screen_recording_permission_required` can be returned when remote control enables Seamless Switching without Screen Recording permission.
- `error`: `null` on success; otherwise an object with a stable `code` and a human-readable `message`.

Error codes:

- `invalid_request`: A required parameter is missing, has the wrong JSON type/value, includes a value with `ToggleSetting`, or tries to toggle a non-Boolean setting.
- `unknown_setting`: The setting name is not recognized.
- `read_only_setting`: The setting can only be changed locally or is derived by APS.
- `unavailable_value`: The requested display or application is not currently available, or the value is a feedback-only fallback such as PDF `none`.

#### APS info
Provides metadata about the running APS instance.
This feedback is sent when a client connects, and again whenever the PC tag is changed in APS preferences.

```json
{
   "action": "aps_info",
   "data": {
      "app_version": "4.2.1.5",
      "computer_tag": "FOH-PC"
   }
}
```
- `app_version`: The APS application version.
- `computer_tag`: The user-defined computer tag on Windows. This maps to the `PC Tag` setting.
- `pc_tag`: The equivalent user-defined computer tag field currently sent by APS for macOS. Clients that support both platforms should accept either `computer_tag` or `pc_tag`.
